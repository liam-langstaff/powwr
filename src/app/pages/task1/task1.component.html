Task

Debug and solve issue regarding the sidebar nav not visible on open toggle. Additionally fix the logo sizing on open and closed state.

Why?

I believe there are a few possible solutions to this. On observation it appears that the the main constraints is coming from the flexbox classes specifically the '.d-flex' and accompying flexbox classes creating constraints resulting in the width being reduced to 0.

Possible solutions:

  • Remove the flex classes entirely. Removing the 'd.flex' and additional flex classes from the sidebar.component.html wrapping div would result in the image expanding to its natural dimensions, outside containers will result in the shrinkage when closing.
  • Adding 'flex-grow: 1' to the :host of branding.component.ts. Adding this flex property will result in the parent growing to its maximum flexbox container width, taking any available space it can. As there is only 1 column, it will fill the full width.

Additional info:

.branding { width: $sidenav-mini - 15px; overflow: hidden; }

It also appears that when the sidenav has been closed, the logo is far too small. On observation, it seems that the width is set to a mini variable in addition to reducing the size by 15px which is not needed in this case. Removing the - 15px makes the logo appear relative to its smaller container.

.branding { padding: 20px; }

Another area which creates reduced logo width is the excessive padding which is applied on both mobile and desktop. Where possible the padding should be reduced to ensure legibility of the logo and prevent further reduction of the logo size. I've created an input to toggle a class when the sidebar is open.
